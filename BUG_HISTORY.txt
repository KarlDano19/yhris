BUG ID: EXAMPLE-001
DATE FILED: 2024-12-15
DATE FIXED: 2024-12-20
CLIENT/REPORTER: QA Team - Maria Santos

ISSUE DESCRIPTION:
Employee sync from HRIS to Payroll was failing when department name contained
special characters (e.g., "R&D Department", "Sales & Marketing"). The sync
process would throw a validation error and skip the employee record silently.

DESIRED OUTCOME:
Employee sync should handle all valid department names including those with
special characters. Any sync errors should be logged and reported clearly.

status component now displays detailed error messages when sync fails,
allowing users to identify and fix problematic records.

BUG ID: EXAMPLE-002
DATE FILED: 2024-12-18
DATE FIXED: 2024-12-22
CLIENT/REPORTER: Systech Internal - Dev Team

ISSUE DESCRIPTION:
Notification bell showing incorrect unread count after marking notifications
as read. The count would only update after page refresh, and sometimes show
negative numbers when rapidly clicking notifications.

DESIRED OUTCOME:
Notification count should update instantly when notifications are marked as
read, with optimistic updates and proper error handling. Count should never
show negative values.

- app/views/notification.py
- app/serializers/notification.py
- tests/views/test_notification.py

Frontend:

- src/hooks/useNotifications.ts

FIX EXPLANATION:

onMutate callback to immediately decrement count before API response. Added
Math.max(0, count) guard in the component's local state to prevent negative
values. Backend now returns updated unread_count in mark-as-read response for
reconciliation. Added debouncing (300ms) to prevent rapid-fire API calls
when clicking multiple notifications quickly. React Query's cache is now
properly invalidated after successful mutation to ensure consistency.
